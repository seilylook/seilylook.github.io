Solidity

seilylook published on 2023-10-19 included in Language
Introduction Solidity 언어 학습 기록. Chapter 1 1.1 Contract Solidity 언어는 Contract 안에 쌓여 있다. Contract는 Ethereum 애플리케이션의 기본적인 구성 요소로, 모든 변수와 함수는 어느 한 Contract 에 속하게 되어 있다. 즉 모든 프로젝트의 시작점이다. Version Pragma 모든 Solidity 소스 코드는 version pragma로 시작해야 한다. 이는 해당 코드가 사용해야 하는 Solidity 버전을 선언하는 것이다. 이를 통해 이후에 새로운 컴파일러 버전이 나와도 기존 코드가 깨지지 않도록 예방하는 것이다. Info npm을 사용한 프로젝트를 개발할 때마다 version 관리 때문에 에러 처리한 경험이 많은데, 위의 기능은 편리하다고 생각된다.

Introduction Web 1.0 | Web 2.0 | Web 3.0은 기술적 차이점도 존재하지만 궁극적으로 웹이 사용되는 방식에따라 구분 짓는다. Web 1.0 | Web 2.0 | Web 3.0 차이점 Web 1.0: World Wide Web의 첫 단계이다. 1991년부터 2004년 동안 우리는 인터넷 1세대를 경험했다. 이 당시는 하이퍼링크를 포함한 커다란 하나의 사이트에 불과했다. 사이트에 무엇을 업로드하면 단순히 보여주기만 하는 기능을 제공했다. 즉 Read-only web이라고 할 수 있다. 사용자와 상호 작용은 전혀 존재하지 않았다. Comments, Authentication, Analytic와 같은 기능이 존재하지 않았다.
