# Web3.0


# Introduction

`Web 1.0` | `Web 2.0` | `Web 3.0`은 기술적 차이점도 존재하지만 궁극적으로 웹이 사용되는 방식에따라 구분 짓는다.

## `Web 1.0` | `Web 2.0` | `Web 3.0` 차이점

- `Web 1.0`: World Wide Web의 첫 단계이다. 1991년부터 2004년 동안 우리는 인터넷 1세대를 경험했다. 이 당시는 하이퍼링크를 포함한 커다란 하나의 사이트에 불과했다. 사이트에 무엇을 업로드하면 단순히 보여주기만 하는 기능을 제공했다. 즉 `Read-only web`이라고 할 수 있다. 사용자와 상호 작용은 전혀 존재하지 않았다. Comments, Authentication, Analytic와 같은 기능이 존재하지 않았다. 단순한 단방향 소통 도구일 뿐이다.

  예시) yahoo

- `Web 2.0`: 2004년부터 현재까지 사용되고 있는 인터넷이다. 이 기간 동안 웹은 많은 것을 포함하게 된다. Web 2.0부터 상호작용, 사회망 연결, 사용자 기반 컨텐츠 등이 포함된다. 상효작용(Interactivity)는 웹에 있어 중요 변화였다. 현재 사용자는 단순히 웹 페이지에서 정보를 소비하는 것을 넘어, 자유롭게 주고받으며 상호작용 할 수 있다. 이때의 웹은 모두 `Centralize` 중앙화 되어 있다. Meta, Google과 같은 사이트는 중앙 서버에서 사용자의 데이터를 모으고, 이를 기반으로 다양한 서비스를 제공해 사용자가 서비스에 더 오래 머무르도록 유도한다. 왜냐하면 오래 머무를수록 돈이 되기 때문이다. 이는 큰 문제를 발생시킨다. 바로 개인정보 탈취의 위험성이다. 이런 문제를 해결하기 위해 Web 3.0이 등장했다.

  예시) Facebook, Google, Twitter

- `Web 3.0`: 가장 큰 특징은 탈중앙화 웹이다. Web 3.0은 중앙서버 없이 peer to peer 네트워크로 연결되어 있다. 모든 회사, 사용자는 정보를 소유할 수 없다. 모든 데이터는 보여지는 동시에 다른 사람들과 연결되어 있다. `Data is shared rather than owned`. 이렇게 함으로써 기존에 발생하던 개인정보 무단 사용, 해킹의 위험으로부터 벗어날 수 있다.

### 결론

1. Web 1.0 -> Read

2. Web 2.0 -> Read, Write (Interaction, Social Media Networks)

3. Web 3.0 -> Read, Write, Own
